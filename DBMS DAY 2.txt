			Types of Keys
1. Super Key
Any combination of one or more attributes 
(columns) that can uniquely identify 
every row (tuple) in a table
Accept NULL VALUES.

  
2. Candidate Key
Minimal set of attributes (columns) that can 
uniquely identify each row (tuple) in a table

3. Primary Key
A primary key is a unique identifier for a specific 
row in a database table.
	

4. Alternate Key
An alternate key is any candidate key in a database 
table that is capable of uniquely identifying a 
record but was not selected as the primary key.


5. Composite Key
A composite key is a database key composed of two 
or more columns that uniquely 
identify a single row in a table

6. Foreign Key
A foreign key is a field (or group of fields) 
in one database table that uniquely identifies 
a row in another table. It establishes 
a link between the two tables, 
enforcing a rule called "referential integrity" 
to ensure that data remains consistent and 
cannot accidentally be deleted or orphaned.



DDL - Data Definition Language
Defines database structure
CREATE, ALTER, DROP, TRUNCATE	
		
DML	- Data Manipulation Language	
Manipulates data in tables	
INSERT, UPDATE, DELETE

DQL	- Data Query Language	
Retrieves data	SELECT

TCL	Transaction Control Language	
Manages transactions	COMMIT, ROLLBACK, SAVEPOINT
DCL	Data Control Language	

Controls access and permissions	GRANT, REVOKE
